# Demo script (10–12 minutes)

1. **Start on the full landscape**
   - Let the graph auto-spin for a few seconds.
   - Explain that distance is based on business purpose, capabilities, features, tech stack, and data/security context.
   - Point out that the axes are latent-style coordinates for the workshop, not literal business axes.

2. **Click Quantexa**
   - Show the dimensions behind the dot.
   - Show the closest neighbors and why they are close.
   - Call out that it sits in the Fraud & Entity Intelligence cluster because of graph analytics, entity resolution, and overlapping Azure stack choices.

3. **Click CVPM Core APAC**
   - Show the pricing / portfolio cluster.
   - Use the nearest approved alternative to demonstrate the approval shortcut.
   - Explain that this is where Alan can quickly identify regional clones or extensions.

4. **Raise the similarity threshold**
   - Show the network getting sparser.
   - Explain that the graph can move from “broad family resemblance” to “strong duplicate candidates.”

5. **Shift the weights**
   - Increase Tech Stack weight to emphasize platform reuse.
   - Increase Business Purpose weight to emphasize use-case duplication.
   - Note that v1 updates the relationships and explanations live while keeping the baseline layout fixed.

6. **Use the source filter**
   - Switch to “SAS-derived only” to show the real extracted sample applications.
   - Switch back to “All apps” to demonstrate the workshop-scale landscape.

7. **End on the review queue**
   - Highlight the recommendation column.
   - Explain that Alan can start from the top potential duplicates and then drill into the clicked node’s evidence pages.
